1. What is fMRI?

Functional Magnetic Resonance Imaging, or fMRI, is a non-invasive way of studying human brain function. Instead of directly recording neurons, most fMRI studies measure the BOLD signal: blood-oxygen-level-dependent changes related to local blood flow, oxygenation and neurovascular coupling.

What it measures

Most fMRI studies measure changes in blood oxygenation. The signal is useful because neural activity changes local energy demand, which changes blood flow and oxygenation.

What it does not measure

It is not a direct measure of action potentials or “thoughts”. Interpretation requires caution because the BOLD response is vascular, delayed, noisy and model-dependent.

Why researchers use it

fMRI can map activity across the whole brain with millimetre-scale spatial resolution while participants perform tasks, rest, watch movies, make decisions, remember, feel emotions or process language.

Key intuition: fMRI is powerful because it lets us study human brain function in vivo, but the result is always a statistical inference from a noisy biological and physical signal. A good fMRI researcher learns both neuroscience and data hygiene.

2. How is an fMRI experiment done?

A typical fMRI project moves from a psychological question to experimental design, scanning, preprocessing, modelling, group statistics and interpretation. Each step can affect the final result.

1QuestionDefine the cognitive process, contrast, hypothesis and expected direction.
2Task designChoose block, event-related, mixed, resting-state or naturalistic design.
3Ethics & safetyScreen for metal, claustrophobia, pregnancy rules, consent and data privacy.
4ScanningAcquire anatomical T1w images, BOLD runs, fieldmaps and behavioural logs.
5PreprocessCorrect motion, align images, normalize to template, smooth, estimate confounds.
6ModelBuild GLM, contrasts, connectivity, MVPA or other analysis depending on question.
7ReportCheck quality, correct for multiple comparisons, share code/maps where possible.

Common task designs

  • Block design: high statistical power; useful when conditions can be sustained for several seconds.
  • Event-related design: flexible and trial-level; useful for randomised cognitive tasks.
  • Mixed design: separates sustained state effects from trial-level responses.
  • Resting-state: participants rest while spontaneous BOLD correlations are analysed.
  • Naturalistic: movies/stories/games; useful for richer real-world cognition.

Core analysis outputs

  • Activation maps: where BOLD signal differs between conditions.
  • Contrasts: statistical tests like Incongruent > Congruent.
  • Connectivity: how regions covary over time or conditions.
  • Decoding/MVPA: whether distributed activity patterns classify mental states or stimuli.
  • ROI analysis: targeted tests in pre-defined brain regions.

3. Main challenges in fMRI research

Good fMRI is not only about running software. It is about understanding what can bias the result and building safeguards into design, preprocessing, statistics and reporting.

Motion

Even tiny head movements can produce structured artefacts, especially in group differences and connectivity analyses.

QC essential

Low signal-to-noise

The BOLD signal is small relative to thermal noise, physiology, scanner drift and participant variability.

Design matters

Indirect signal

BOLD is related to neurovascular coupling, so it should not be interpreted as a direct readout of neuronal firing.

Interpret carefully

Multiple comparisons

Whole-brain maps test thousands of voxels. Without correction, false positives become likely.

Statistics essential

Flexible pipelines

Different preprocessing, smoothing, denoising, template and threshold choices can change results.

Pre-register when possible

Reverse inference

Seeing activation in a region does not prove a specific mental process unless the task and evidence support that inference.

4. Skills students should build

You do not need to master everything before starting. But you should gradually build enough skill to understand the full pipeline rather than treating fMRI analysis as a black box.

5. A practical 8-week learning path

This is a realistic route for a psychology, neuroscience, biology or cognitive science student who wants to start working with fMRI data.

Week 1

Understand the big picture

Watch an introductory fMRI lecture series. Learn what BOLD is, what a contrast is, and why design matters.

Week 2

Learn the command line

Practise folders, paths, shell commands, loops, permissions and simple scripts. This removes the biggest barrier for beginners.

Week 3

Open real MRI files

Download a small OpenNeuro dataset. View T1w and BOLD images in FSLeyes, AFNI or Nilearn. Learn NIfTI, TR, voxel size and image orientation.

Week 4

Learn BIDS

Understand subject/session/run folders and JSON metadata. Run the BIDS Validator and fix basic errors.

Week 5

Run preprocessing

Read fMRIPrep/MRIQC reports. Learn motion correction, registration, normalization, confounds and why visual QC is non-negotiable.

Week 6

Run a task GLM

Use Andy’s Brain Book, FSL, SPM or AFNI to model a simple task dataset. Inspect the design matrix and contrasts.

Week 7

Try one advanced direction

Choose one: resting-state connectivity, ROI analysis, MVPA/decoding, surface analysis, or naturalistic analysis.

Week 8

Make a reproducible mini-project

Create a short report with research question, data source, methods, QC screenshots, results, limitations and code link.

7. Hands-on mini-project ideas

The best way to enter neuroimaging is to finish a small analysis end-to-end. These projects are realistic for a student portfolio or lab application.

Project A: First fMRI dataset tour

  1. Download a small task-fMRI dataset from OpenNeuro.
  2. Inspect anatomical and functional files in FSLeyes/AFNI.
  3. Identify TR, voxel size, number of volumes and task events.
  4. Write a one-page “what is in this dataset?” report.

Project B: BIDS + validation

  1. Take a tiny sample dataset or BIDS example.
  2. Learn the folder naming structure.
  3. Run BIDS Validator.
  4. Fix or document warnings and errors.

Project C: Simple task GLM

  1. Use a Flanker, motor, n-back or emotional task dataset.
  2. Run one first-level GLM using Andy’s Brain Book/FSL/SPM/AFNI.
  3. Make one contrast map and one design matrix figure.
  4. Explain the contrast in plain language.

Project D: QC report

  1. Run or inspect fMRIPrep/MRIQC reports.
  2. Identify motion, registration and signal dropout issues.
  3. Make a table of accepted/excluded runs.
  4. Write transparent exclusion criteria.

Project E: Functional connectivity

  1. Use resting-state or task residual data.
  2. Extract seed time series from an ROI.
  3. Correlate with other regions or whole brain.
  4. Report why denoising and motion control matter.

Project F: Decoding/MVPA

  1. Use Haxby-style visual category data or another labelled task.
  2. Train a classifier with Nilearn or TDT.
  3. Use cross-validation correctly.
  4. Report accuracy, chance level, leakage risks and limitations.

8. Beginner’s checklist before joining an fMRI project

You do not need to tick every box to start, but each item will make you more useful in a neuroimaging lab.

I can explain BOLD in simple language without saying it directly measures thoughts.
I know the difference between anatomical MRI and functional MRI.
I understand what a task contrast is.
I can navigate folders and run basic Linux commands.
I can open a NIfTI file and inspect image orientation/slices.
I know what BIDS is and why it matters.
I understand motion correction, registration and normalization at a basic level.
I can read a design matrix and ask whether regressors are sensible.
I know why multiple-comparison correction matters.
I can write a clear README for data, code and outputs.
Suggested order for absolute beginners: MIT fMRI Bootcamp → Software Carpentry/Missing Semester → Andy’s Brain Book → OpenNeuro dataset → BIDS Validator → fMRIPrep/MRIQC reports → one simple GLM → one mini-report.

9. Tiny glossary

BOLD

Blood-oxygen-level-dependent signal; the most common fMRI contrast.

Voxel

A 3D pixel: the smallest volume element in an MRI image.

TR

Repetition time: how often a whole brain volume is acquired.

HRF

Hemodynamic response function: model of the delayed BOLD response after neural events.

GLM

General Linear Model: the standard modelling framework for task-fMRI activation analysis.

Contrast

A statistical comparison, often between conditions, such as Reward > No Reward.

ROI

Region of interest: a predefined anatomical or functional brain region.

MVPA

Multi-voxel pattern analysis: decoding information from distributed patterns of activity.

BIDS

A standard structure for organising brain imaging data and metadata.
